真实项目中的前后端完全分离开发流程:
[产品部门]
  产品:一个产品或者而一个项目从最初的IDEA到有最后的产品，都是产品人员去做的
  UI:把产品设计的原型通过合理的排版和颜色搭配，设计出项目的“效果图 PSD格式的”

  ->需求、功能、效果图等都有了

[技术部]
  前端：首先拿到效果图(PSD),使用PS把需要的尺寸颜色图片等都切下来(切图); 使用HTML(5)+CSS(3)搭建基本的布局结构,使用JS或者类库实现基础的交互功能; 前端开发通过AJAX或者JSONP向服务器发送请求把需要的数据从服务器获取到，然后动态的展示在HTML页面上（向服务器发送请求需要有一个请求的地址，这个地址我们称之为API接口，请求的接口都是后台人员提供给我们的，项目中会有API文档）；

  后端：根据需求完成所需要的所有的功能，并且把每一个功能都提供一个对应的API接口，编写接口文档，把接口文档给前端；

  测试：前后端开发完成后，提交测试，测试人员开始进行黑盒或者白盒测试

  服务器运营：产品测试通过后，服务器运营人员把项目进行发布，小公司的后台就是服务器运营人员

[运营推广部门]
  SEO
  SEM
  ...


API文档:接口文档，前后端开发的规范，前端按照接口文档调取，服务器端按照接口文档实现对应的功能即可 Application Programming Interface
客户端和服务器端数据传输所使用的大部分都是JSON格式的字符串

1、展示所有的客户信息
   URL:/getAllList  GET
   参数:无
   返回:
   '{
     code:0,//->0成功 1失败
     msg:'',//->对CODE的描述
     data:[
        {
          id:1,
          name:'xxx',
          age:24,
          phone:'',
          address:''
        },
        ...
     ]
   }'

2、增加客户信息
   URL:/addInfo POST
   参数:'{name:'xxx',age:25,phone:'',address:''}'
   返回:
   '{
      code:0,
      msg:''
   }'

4、修改客户信息
  URL:/updateInfo POST
  参数:'{id:1,name:'',age:24,phone:'',address:''}'
  返回:
  '{
     code:0,
     msg:''
  }'

3、获取指定客户的信息
   URL:/getInfo GET
   参数:?id=1 传递给服务器对应的客户的ID
   返回:
   '{
      code:0,
      mag:'',
      data:{
         id:1,
         name:'',
         age:24,
         phone:'',
         address:''
      }
   }'




5、删除客户信息
   URL:/removeInfo GET
   参数:?id=1
   返回:
   '{
     code:0,
     msg:''
   }'
